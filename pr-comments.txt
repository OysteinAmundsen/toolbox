---thread 14 id=PRRT_kwDOQtIN_859e-el resolved=false file=libs/grid/src/lib/plugins/row-drag-drop/RowDragDropPlugin.ts
  [copilot-pull-request-reviewer]: `onKeyDown()` uses `grid._rows ?? this.sourceRows` to determine the focused row, but `canMoveRow()` always reads from `this.sourceRows`. During debounced keyboard moves, `_rows` diverges from `sourceRows`, so `canMoveRow()` can end up validating a different row than the one being moved (and can call `canDrag`/`canMove`/queries with the wrong row). Please have `canMoveRow()` resolve rows from `this.internalGrid._rows ?? this.sourceRows` (or accept the current row array as a parameter) so validation matches the indices used by keyboard moves.
